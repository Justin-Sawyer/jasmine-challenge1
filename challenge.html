Red-Green-Refactor

Develop code incrementally.

Start by writing code that fails (red!), then just enough to make it pass (green!).

The challenge:

Drink About is a simple function that will check the age and return a drink appropriate for your age. Create a function called whatCanIDrink. This function should take in age as parameter. The function should do the following:


If the age is less than 0, then it should return "Sorry. I can’t tell what drink because that age is incorrect!"

If the age is less than 14, then it should return "Drink Toddy"

If the age is less than 18, then it should return "Drink Coke"

If the age is less than 21, then it should return "Drink Beer"

If the age is less than 130, then it should return "Drink Whisky"

Else return "Sorry. I can’t tell what drink because that age is incorrect!"


What am I testing for?

Am I testing for age?

Yes... but age is what decides what I can drink.

Am I testing for the drink?

Yes. The drink I can consume is a result of what age I am.


Thus, the function should start with asking what I can drink, and have age as its parameter.

What can I drink? -> That depends on my age -> If I am this age, I can drink that.

So, the parameter "age" must be filled in. It must not return "undefined"


Programming defensively

What can break the code?

If the age value is not a Number, the code should return an alert.


What if the age is less than 0 or greater than 130?

These are values of the age parameter. A result should be returned, but not as an Error.
